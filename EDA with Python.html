Exploratory Data Analysis (EDA) with Python¶

January 2019

Melvin Ott, PhD

Linear regression model building begins with a thorough look at the quality of the data and focuses on how to repair faulty data. Students are given direction with Python code on how to find and repair this faulty data. Some students follow this and provide very good repair while others overlook this very important step. To help these students that overlook the repair step, I have drafted this example which will show model performance improvement with increasing levels of data repair. The repair process fits within what is called Exploratory Data Analysis.

A question often asked is, "How do I do EDA?" The answer is not a simple one because data is often collected in a haphazard way or may be corrupted by unknown sources. Or, in some cases it may even be the wrong data.

Missing data, finding and fixing records that may be duplicates, out of range or outlier values, and issues that are application specific all present a challenge.

Keep in mind there are as many unknown data issues as there are ways to make mistakes. Not all are discoverable. But the best way to find them is to look at each dataset as a new challenge.

For example, health care claims data has issues that are application specific. There are code combinations that are denied for payment for many different reasons. There may be duplicate claims and claims that are for noncovered services, billed to wrong insurance and many more. When in doubt work with paid claims. Paid claims will remove most of these issues but some duplicate claims may still be paid.

So, the answer is dependent on the application and the best EDA will likely be performed by an individual that has knowledge of the industry and has the experience to deal with suspect data.

For this example a few of these issues are addressed in a fairly simple dataset. These are missing values, duplicate records, and out of range values. But the process described should be transferable to other datasets.

Finally, after the EDA, a scatter plot with color for a third variable suggests a more complex model. An engineer told me once that you cannot present data with more than two variables in two dimensions. He was wrong. You can do it with color and you can do it with size for the plot tick marks.

Section 2 Base Model and RMSE¶

Run a base model and calculate the root mean square error for the model. Kaggle is set up to score your models using RMSE and my score code also uses RMSE. However, I have penalty's for out of range predictions and missing predictions so make sure none of these will apply to your model predictions.

Using the statsmodels API for this linear model and taxes for the single prediction variable. Taxes will be used for the single prediction variable for each of the sections where the change will be improvement in the data due to the EDA. Result will be model RMSE improvement due to the EDA rather than changes to the input variable.

In [86]:

d = {'p_price': predictions}

join['sq_error'] = (join['price'] - join['p_price'])**2 
mean_sq_error =  np.sqrt(join.sq_error).mean()
BASE_RMSE = np.around([mean_sq_error], decimals=2)
